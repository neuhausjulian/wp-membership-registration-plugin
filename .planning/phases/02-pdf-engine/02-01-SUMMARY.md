# Phase 2 Plan 01: DOMPDF + PHPUnit Scaffold Summary

**One-liner:** DOMPDF 3.1.5 installed and namespace-scoped under `WpMembershipRegistration\Vendor\Dompdf\` via Strauss; PHPUnit 9.6 + Brain Monkey 2.7 configured with 3-stub test scaffold.

## What Was Built

Wave 0 infrastructure for the PDF engine:

1. **DOMPDF scoped under Strauss** — `vendor-prefixed/autoload.php` provides collision-free DOMPDF classes for the plugin, preventing conflicts with other plugins that may bundle DOMPDF.

2. **PHPUnit test scaffold** — `phpunit.xml` configures the `unit` testsuite; `tests/bootstrap.php` loads both autoloaders without WordPress; `tests/Unit/Pdf/PdfGeneratorTest.php` contains 3 stub methods (PDF-01, PDF-02, PDF-03) that mark incomplete and exit 0.

3. **Main plugin file updated** — `vendor-prefixed/autoload.php` is conditionally required before `Plugin::register()`, with a `file_exists()` guard that prevents fatal errors on fresh clones.
