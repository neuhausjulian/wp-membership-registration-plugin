# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Plugin Foundation and Settings.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Context Note

This phase produces a **WordPress admin settings page**, not a React/Next.js/Vite frontend. The design system is the WordPress admin UI itself. shadcn is not applicable. All visual decisions are governed by:

1. WordPress admin conventions (`wp-admin` CSS, standard patterns)
2. A small plugin-scoped stylesheet (`assets/css/admin-settings.css`) for the field schema editor and custom elements
3. Bundled WP libraries only — no external JS or CSS downloads

No `components.json`. No Tailwind. No shadcn. Registry safety gate: not applicable.

## Spacing Scale

All spacing values used in `assets/css/admin-settings.css` must come from this scale (multiples of 4px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, drag handle padding, inline badge spacing |
| sm | 8px | Table cell padding (horizontal), button gap from adjacent element |
| md | 16px | Table cell padding (vertical), form field label-to-input gap, tab panel top padding |
| lg | 24px | Section heading margin-bottom, tab panel side padding |
| xl | 32px | Gap between tab panel and page heading |
| 2xl | 48px | Not used in Phase 1 |
| 3xl | 64px | Not used in Phase 1 |

Exceptions:

- Touch/click targets for the drag handle and delete button: minimum 44px tall (WP admin accessibility standard — aligns to 44px row height in the field schema table)
- WP admin standard `#wpcontent` padding (20px) is inherited and must not be overridden

---

## Typography

All type is inherited from WordPress admin CSS. The plugin stylesheet must **not** redeclare font-family, font-size, or line-height for body text or standard form labels. The following roles apply only to plugin-specific UI elements that deviate from the WP admin baseline.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body / form label | 14px (inherited) | 400 (inherited) | 1.5 (inherited) | All standard settings labels and textarea content |
| Table column header | 14px | 600 (semibold) | 1.2 | Field schema editor table `<th>` cells |
| Section heading | 20px (inherited `h2`) | 600 (inherited) | 1.2 (inherited) | Tab section headings rendered by WP `add_settings_section()` |
| Small / helper text | 12px | 400 | 1.5 | Inline description text below inputs (WP `.description` class) |

Declared sizes: 12, 14, 20 (3 sizes — inherited; plugin CSS only overrides where WP default is absent)
Declared weights: 400 (regular) + 600 (semibold)

## Color

WordPress admin applies the active admin color scheme via CSS custom properties. The plugin CSS must not hardcode WP chrome colors. The plugin-specific color tokens below are for elements that have no WP admin equivalent.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#f0f0f1` (WP admin background) | Page background, tab panel background | WP admin default — inherited, do not override |
| Secondary (30%) | `#ffffff` | Settings form area background, table background | WP admin `.postbox` / `.wrap` convention |
| Accent (10%) | `#2271b1` (WP admin primary blue) | Active nav-tab indicator, Save button (rendered by WP `submit_button()`) | WP admin button class — inherited via `.button-primary` |
| Destructive | `#d63638` | Delete field row button only | WP admin `--wp-admin-theme-color-darker-10` destructive convention |
| Neutral / drag handle | `#8c8f94` | Drag handle icon color (`dashicons-menu`) | WP admin `.description` text color |

Accent reserved for: active tab underline indicator, primary "Save Changes" button (via WP `.button-primary` — do not apply accent to any other element in plugin CSS).

Destructive (`#d63638`) reserved for: the "Delete" button in each field schema row only.

## Component Inventory

Specific HTML patterns and their exact attributes for Phase 1. Executor must use these patterns verbatim.

### Tab Navigation

```html
<nav class="nav-tab-wrapper wmr-tab-nav">
    <a href="#wmr-tab-fields"   class="nav-tab nav-tab-active" data-tab="fields">Form Fields</a>
    <a href="#wmr-tab-branding" class="nav-tab"                data-tab="branding">PDF Branding</a>
    <a href="#wmr-tab-email"    class="nav-tab"                data-tab="email">Email Settings</a>
</nav>
```

- Active tab: class `nav-tab-active` on the visible panel's anchor
- Inactive panels: `style="display:none"` set by JS on page load (not by PHP, to allow JS-free degradation)
- Tab switch: JS removes `nav-tab-active` from all, adds to clicked anchor, hides all `.wmr-tab-panel`, shows the targeted panel

### Field Schema Table

```html
<table class="wp-list-table widefat fixed striped wmr-fields-table">
    <thead>
        <tr>
            <th style="width:40px"></th>          <!-- drag handle column -->
            <th>Label</th>
            <th style="width:160px">Field Type</th>
            <th style="width:80px">Required</th>
            <th style="width:60px"></th>           <!-- delete column -->
        </tr>
    </thead>

    </tbody>

```

Each row structure:

```html
</tr>
```

### Send Test Email Button (Email Settings tab)

```html
<button type="button" id="wmr-send-test-email" class="button">
    <span class="dashicons dashicons-email" aria-hidden="true" style="margin-top:3px;"></span>
    Send Test Email
</button>
<span id="wmr-test-email-result" class="wmr-inline-notice" aria-live="polite"></span>
```

Interaction states:
- Default: enabled, default button style
- In-flight (AJAX pending): disabled + button text becomes "Sending…" (no spinner graphic — text only)
- Success: `wmr-inline-notice--success` class on the result span, text "Test email sent."
- Error: `wmr-inline-notice--error` class on the result span, text from `wp_send_json_error` response

## Interaction States

Every interactive element must implement all declared states.

| Element | Default | Hover / Focus | Active / In-flight | Success | Error |
|---------|---------|---------------|--------------------|---------|-------|
| Save Changes button | `.button-primary` (WP styled) | WP default hover | Disabled during submit (form post, page reloads — browser handles) | WP admin success notice at top of page | WP admin error notice at top of page |
| Send Test Email button | `.button` enabled | WP default hover | Disabled, text "Sending…" | Inline notice "Test email sent." in green | Inline notice with error message in red |
| Delete row button | `.button` enabled | WP default hover | Row fades out and is removed from DOM | n/a | n/a |
| Add Field button | `.button` enabled | WP default hover | New row appended to tbody | n/a | n/a |
| Nav tabs | `.nav-tab` inactive | WP default hover | `.nav-tab-active` applied | n/a | n/a |
| Drag handle | cursor default | `cursor:grab` on hover | `cursor:grabbing` while dragging | n/a | n/a |

## Accessibility Notes

These are requirements, not suggestions.

- All `<button>` elements that contain only an icon must have a visible text label or an `aria-label` attribute.
- The recipient textarea must have `aria-describedby` pointing to its description paragraph.
- The test email result span must have `aria-live="polite"` so screen readers announce the outcome.
- The drag-and-drop reorder is a progressive enhancement. If JS is unavailable, the table rows still render and the save button still submits (fields will be saved in their current order without reordering capability).
- All Dashicons used for interactive purposes must have `aria-hidden="true"` on the `<span>` with a text label or `aria-label` on the parent button.
- Color contrast: all text elements must meet WCAG 2.1 AA (4.5:1 for normal text, 3:1 for large text). WordPress admin color tokens already meet this threshold; do not introduce custom colors that bypass it.
