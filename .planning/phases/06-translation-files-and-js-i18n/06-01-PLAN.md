**PluginTest.php changes:**

Add a new test method after the existing Test 6. This test verifies that the wp_enqueue_scripts callback (which is registered inside Plugin::register()) calls `wp_set_script_translations` for the `wmr-form` handle.

The challenge: The wp_enqueue_scripts callback is an anonymous closure registered via `add_action('wp_enqueue_scripts', ...)`. To test it, we need to capture it similar to how init is captured.

Add a new private helper and test:

```php
/** @var callable|null Captured wp_enqueue_scripts callback. */
private $captured_enqueue_callback = null;

/**
 * Register Plugin hooks and capture the 'wp_enqueue_scripts' callback.
 */
private function register_plugin_and_capture_enqueue(): void {
    $self = $this;

    Functions\when( 'add_action' )->alias(
        function ( $hook, $cb ) use ( $self ) {
            if ( 'wp_enqueue_scripts' === $hook ) {
                $self->captured_enqueue_callback = $cb;
            }
        }
    );

    Functions\stubs(
        [
            'add_shortcode'      => true,
            'get_option'         => fn( $k, $d = null ) => $d,
            'wp_kses_post'       => fn( $v ) => (string) $v,
            'wp_enqueue_style'   => true,
            'wp_enqueue_script'  => true,
            'wp_localize_script' => true,
            'wp_create_nonce'    => 'test_nonce',
            'admin_url'          => fn( $p = '' ) => 'http://example.com/wp-admin/' . $p,
            'plugin_basename'    => fn( $f ) => 'wp-membership-registration/wp-membership-registration.php',
            'load_plugin_textdomain' => true,
            'is_singular'        => true,
            'is_a'               => true,
            'has_shortcode'      => true,
        ]
    );

    ( new Plugin() )->register();
}

/**
 * Test 7: wp_enqueue_scripts callback calls wp_set_script_translations for wmr-form.
 */
public function test_enqueue_calls_set_script_translations_for_form(): void {
    $this->register_plugin_and_capture_enqueue();

    $this->assertNotNull( $this->captured_enqueue_callback, 'No callback registered for wp_enqueue_scripts.' );

    // The callback checks global $post and has_shortcode — mock those.
    // Already stubbed is_singular, is_a, has_shortcode as true above.
    // We need $post global.
    $GLOBALS['post'] = (object) [ 'post_content' => '[membership_form]' ];

    Functions\expect( 'wp_set_script_translations' )
        ->once()
        ->with( 'wmr-form', 'wp-membership-registration', \Mockery::on( fn( $p ) => str_ends_with( $p, 'languages' ) ) );

    call_user_func( $this->captured_enqueue_callback );

    unset( $GLOBALS['post'] );
}
```

Note: The `wp_enqueue_scripts` callback in Plugin.php checks `is_singular()`, `is_a( $post, 'WP_Post' )`, and `has_shortcode()`. The stubs must return truthy values for these. The `$GLOBALS['post']` mock provides the `$post` global.

If `is_a` needs to be more specific (Brain Monkey may not stub built-in `is_a`), use `Functions\when( 'is_a' )->justReturn( true )` or set `$GLOBALS['post']` to be a mock that passes the check. Test carefully -- if Brain Monkey cannot stub `is_a()` (a PHP builtin), then create a simple stdClass with the right properties and rely on the function stubs for `is_singular` and `has_shortcode`.
