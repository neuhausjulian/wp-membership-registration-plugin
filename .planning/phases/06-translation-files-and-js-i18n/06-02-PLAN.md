**Step 1: Ensure languages/ directory exists**

```bash
mkdir -p languages
```

**Step 2: Generate .pot file using WP-CLI**

```bash
docker run --rm \
  --user "$(id -u):$(id -g)" \
  -v "$(pwd):/var/www/html/wp-content/plugins/wp-membership-registration" \
  wordpress:cli \
  wp i18n make-pot \
    /var/www/html/wp-content/plugins/wp-membership-registration \
    /var/www/html/wp-content/plugins/wp-membership-registration/languages/wp-membership-registration.pot \
    --domain=wp-membership-registration \
    --exclude=vendor,vendor-prefixed,node_modules,tests,.planning,bin
```

After running, verify the .pot file exists and contains expected strings:
- `grep 'Please confirm your consent' languages/wp-membership-registration.pot`
- `grep 'Drag to reorder' languages/wp-membership-registration.pot`

**Step 3: Author the de_DE .po file**

Create `languages/wp-membership-registration-de_DE.po` with:

1. Standard .po header (see template below)
2. One `msgid`/`msgstr` pair for EVERY entry in the .pot file

Use the .pot file as the source of truth for msgids. Copy each msgid exactly (byte-for-byte) and provide the German translation.

**Translation guidelines (from user decisions):**
- Informal "du" tone everywhere (not "Sie")
- Reuse original German phrasings where they existed before Phase 5 English conversion
- All strings must have a non-empty msgstr (no untranslated entries)

**Step 4: Compile .po to .mo**

```bash

  "apt-get update -qq && apt-get install -y --no-install-recommends gettext && \
   msgfmt /var/www/html/wp-content/plugins/wp-membership-registration/languages/wp-membership-registration-de_DE.po \
          -o /var/www/html/wp-content/plugins/wp-membership-registration/languages/wp-membership-registration-de_DE.mo"
```
