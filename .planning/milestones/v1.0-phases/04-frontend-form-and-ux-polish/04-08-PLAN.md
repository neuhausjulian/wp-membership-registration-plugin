From src/Pdf/PdfGenerator.php (lines 124-128), the current form_notes block:
```php
// --- form_notes block (HTML from TinyMCE) ---
if ( $form_notes ) {
    $pdf->Ln( 4 );
    $pdf->writeHTML( $form_notes, true, false, true, false, '' );
}
```

The page2_content block (lines 148-153) works correctly because AddPage() resets TCPDF state:
```php
if ( $page2_content ) {
    $pdf->AddPage();
    $pdf->SetFont( 'dejavusans', '', 10 );  // <-- SetFont before writeHTML
    $pdf->writeHTML( $page2_content, true, false, true, false, '' );
}
```

Root cause: writeHTML() for form_notes is called mid-page after Cell()/Ln() operations. TCPDF's block-level tag handling requires a clean font state. The fix mirrors the page2_content pattern: call SetFont() before writeHTML().
