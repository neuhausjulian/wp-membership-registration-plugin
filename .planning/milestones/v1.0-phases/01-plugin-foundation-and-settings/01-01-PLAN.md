Create `.phpcs.xml` with the following exact content:

```xml
<?xml version="1.0"?>
<ruleset name="WP-Membership-Registration">
    <description>PHPCS rules for WP Membership Registration plugin.</description>

    <!-- Scan src/ and the main plugin file only -->
    <file>src/</file>
    <file>wp-membership-registration.php</file>

    <!-- Exclude generated files -->
    <exclude-pattern>vendor/</exclude-pattern>
    <exclude-pattern>node_modules/</exclude-pattern>

    <!-- Use the full WordPress standard -->
    <rule ref="WordPress">
        <!-- Allow PSR-4 PascalCase filenames (class-*.php not required) -->
        <exclude name="WordPress.Files.FileName.NotHyphenatedLowercase"/>
        <exclude name="WordPress.Files.FileName.InvalidClassFileName"/>
    </rule>

    <!-- Set text domain for i18n sniff -->
    <rule ref="WordPress.WP.I18n">
        <properties>
            <property name="text_domain" type="array">
                <element value="wp-membership-registration"/>
            </property>
        </properties>
    </rule>

    <!-- PHP version minimum -->
    <config name="minimum_supported_wp_version" value="6.0"/>
    <config name="testVersion" value="8.0-"/>
</ruleset>
```

Create the directory `dev/mu-plugins/` and create `dev/mu-plugins/mailpit-smtp.php` with the following exact content:

```php
<?php
/**
 * MU-Plugin: Route wp_mail() through Mailpit for local development.
 *
 * Loaded automatically by WordPress from mu-plugins/. Only mounted
 * in the Docker environment — not included in production plugin zip.
 *
 * @package WpMembershipRegistration
 */

// If this file is called directly, abort.
if ( ! defined( 'ABSPATH' ) ) {
	exit;
}

add_action(
	'phpmailer_init',
	function ( $phpmailer ) {
		$phpmailer->isSMTP();
		$phpmailer->Host       = 'mailpit';
		$phpmailer->Port       = 1025;
		$phpmailer->SMTPAuth   = false;
		$phpmailer->SMTPSecure = '';
	}
);
```

Note: The `WORDPRESS_CONFIG_EXTRA` approach for SMTP is NOT used (it does not hook phpmailer_init — the mu-plugin approach is the correct mechanism as documented in RESEARCH.md Pitfall 6). The SMTP redirect is handled entirely by the mu-plugin.
