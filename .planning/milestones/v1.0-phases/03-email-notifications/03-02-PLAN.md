    File header:
    ```php
    <?php
    /**
     * Mailer — sends membership registration emails with PDF attachment.
    ```

    Class: `Mailer`

    Constructor:
    ```php
    /**
     * @param PdfGenerator|null $generator Optional PdfGenerator instance (injected for testing).
     */
    public function __construct( private ?PdfGenerator $generator = null ) {}
    ```

    Public method `handle_submission(array $field_values, string $member_email): void`:
    - Instantiate `$this->generator ?? new PdfGenerator()` and call `->generate($field_values)` to get `$pdf_path`
    - Wrap all sending logic in try/finally:
      ```php
      try {
          $this->send_member_email($member_email, $pdf_path);
          $this->send_admin_email($pdf_path);
      } finally {
          wp_delete_file($pdf_path);
      }
      ```

    Private method `send_member_email(string $member_email, string $pdf_path): void`:
    - Resolve the member address: scan FieldSchema::decode(get_option('wmr_field_schema', '[]')) for first field with type === 'email'; if found, use `sanitize_email($field_values[$field['label']] ?? '')` from the submitted values
    - BUT: the hook already passes `$member_email` as the second parameter — use it directly (it was already resolved by the caller). Check if `$member_email` is a non-empty, valid email via `is_email($member_email)`.
    - If `!is_email($member_email)`: return early (do not send)
    - If valid: call wp_mail() with:
      - $to: $member_email
      - $subject: `__('Your registration form', 'wp-membership-registration')`
      - $message: `__('Thank you for your registration. Please find your completed membership form attached.', 'wp-membership-registration')`
      - $headers: `array()` (empty — use site defaults, plain text)
      - $attachments: `array($pdf_path)`
    - Capture return value; if false: `error_log('wmr: wp_mail() failed sending member confirmation to ' . $member_email)`

    Private method `send_admin_email(string $pdf_path): void`:
    - Parse admin recipients using the canonical pattern from AjaxHandlers.php:
      ```php
      $settings   = get_option('wmr_email_settings', array());
      $raw        = $settings['recipients'] ?? '';
      $recipients = array_values(
          array_filter(
              array_map(
                  'sanitize_email',
                  explode("\n", $raw)
              )
          )
      );
      ```
    - If empty($recipients): return early
    - Call wp_mail() with:
      - $to: $recipients (array)
      - $subject: `__('New membership registration received', 'wp-membership-registration')`
      - $message: `__('A new membership registration was submitted. The completed form is attached.', 'wp-membership-registration')`
      - $headers: `array()`
      - $attachments: `array($pdf_path)`
    - Capture return value; if false: `error_log('wmr: wp_mail() failed sending admin notification to ' . implode(', ', $recipients))`

    WPCS compliance requirements:
    - Use `wp_delete_file()` not bare `unlink()`
    - Use `wp_mail()` not raw `mail()`
    - All string literals wrapped in `__()` with text domain 'wp-membership-registration'
    - DocBlocks on all methods
